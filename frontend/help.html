Help & User Manual

This page explains how Taskerrand works and how to use the site features. It is written for users and administrators. If something on the live site behaves differently, try clearing cache or contact the project owner.

Getting Started

  • Create an account / Sign in: Sign in using the site authentication (Firebase). Use the Login/Sign-in button on the landing page.
  • Profile: After signing in, your name and profile picture appear on the top-right. Click the profile image to view your dashboard or account details.

Navigation

  • Dashboard: Shows your tasks and quick stats.
  • Help: This page.
  • Browse Tasks: Public list of tasks you can accept.
  • Post Task: Create a new task as a creator.
  • Admin: Visible only to admins. Access admin tools like reported tasks.
  • Notifications: Click the bell icon to view notifications. Use the "View" button to open related task, or "Delete" to remove the notification.

Posting a Task (as Creator)

  1. Open Post Task from the nav.
  2. Fill in required fields: title, description, payment, contact number, schedule and location. The map helps pick latitude/longitude.
  3. Submit the form to post the task. Newly posted tasks have status available.
  4. To cancel a task you posted: go to your Dashboard > My Tasks and click the Cancel action for tasks that are still available.

Accepting and Completing Tasks (as Seeker)

  • From Browse Tasks, open a task and click Accept to take it. Accepted tasks become ongoing.
  • When the work is done, the seeker marks the task Complete. The creator can then confirm completion.
  • Statuses may include: available, ongoing, pending_confirmation, completed, cancelled.

Reporting a Task

If you see an inappropriate or problematic task you can report it. Use the report flow and choose one of these types:

  • Fraudulent: Task is a scam or request for payment schemes.
  • Illegal: Task requests or encourages illegal activity.
  • Inappropriate: Offensive, abusive, or otherwise unacceptable content.
  • Other: Any other reason‚Äîprovide details in the description.

Admins will review reported tasks and may remove tasks or dismiss reports.

Feedback and Ratings

After completing a task, users can leave feedback and ratings for other users. This helps build trust in the system.

Notifications

  • Notifications are shown via the bell icon in the header.
  • Each notification may reference a task. Use "View" to open the task. Use "Delete" to remove the notification.
  • Timestamps are shown in your browser locale/timezone.

Admin Functionality

  • Admin Dashboard: Admins can view and filter all tasks and reports.
  • Reported Tasks: Admins can view reports, delete the associated task, or dismiss a report.
  • User Management: Admins can view all users.
  • Only users flagged as admins in the backend see the admin links and pages.

Data & Deletion Notes

  • Deleting a task will also remove related messages, feedback, reports and notifications to keep data consistent.
  • Only task posters or admins can delete a task through the normal UI; admins have a special delete endpoint for moderation.

Troubleshooting

  • If a page seems stuck loading, open the browser developer console (F12) and check the Network tab and console errors.
  • If your notifications times look wrong, refresh the page‚Äîtimestamps are shown in your device's local timezone.
  • For authentication issues, ensure you are signed in and your token is valid (login again if needed).
